# 周K线选股策略 - 使用指南

## 📋 方案概览

### 方案A（稳健型）✅ 推荐

**特点**：
- 使用上周的完整周K数据（数据准确，无预测误差）
- 五步流程筛选股票
- 可选AI分析和新闻整合

**适用场景**：
- 追求稳健、准确的选股结果
- 不希望引入预测误差
- 适合中长期投资

**权重分配**：
- 无需权重分配（使用完整历史数据）

---

### 方案B（激进型）⚡ 高风险

**特点**：
- 第五步：只筛选价格和成交额
- 第六步：获取实时日K数据，拟合本周周K
- 权重分配：周一高权重，逐渐下降，周五不给权重
- 根据成交量均线和偏离度筛选

**适用场景**：
- 追求更及时的选股信号
- 愿意承担预测误差的风险
- 适合短线交易

**权重分配**：
```
周一: 40%  |  周二: 30%  |  周三: 20%  |  周四: 10%  |  周五: 0%
```

**拟合策略（改进版 - 偏离值拟合）**：
- ❌ 价格指标（开盘价、最高价、最低价、收盘价）：使用实时数据，不拟合
- ❌ 成交量：使用本周累计成交量，不拟合
- ❌ 成交额：不拟合（第五步已经筛选过）
- ✅ 5周成交量均线：根据本周累计成交量重新计算
- ❌ 60周成交量均线：不变（周期太长）
- ✅ 偏离值：根据新的5周成交量均线重新计算

**为什么这样设计？**
1. **避免权重分配问题**：不拟合成交量，直接使用本周累计值
2. **逻辑清晰**：价格和成交量都是实时数据，无预测误差
3. **符合实际**：如果本周成交量放大，直接反映在5周成交量均线中

## 🔧 技术细节

### 数据源故障切换机制

两个方案都遵循多数据源故障切换机制：

1. **股票列表获取**：6个数据源故障切换
   - Akshare → Efinance → Baostock → Tushare → Pytdx → Yfinance

2. **周K数据获取**：6个数据源故障切换
   - Akshare → Efinance → Baostock → Tushare → Pytdx → Yfinance

3. **日K数据获取**（仅方案B）：多数据源故障切换
   - 使用 `DataFetcherManager.get_daily_data()` 自动切换

### 周K拟合算法（仅方案B）

**拟合公式**：
```
fitted_value = last_week_value * (1 - weight) + current_week_value * weight
```

**权重分配逻辑**：
- 周一：40% 权重（市场情绪最强烈）
- 周二：30% 权重
- 周三：20% 权重
- 周四：10% 权重
- 周五：0% 权重（使用上周完整数据，不拟合）

**为什么这样分配权重？**
- 周一：经过周末的信息消化，市场情绪最强烈，权重最高
- 周二到周四：市场逐渐稳定，权重逐渐下降
- 周五：本周即将结束，使用上周完整数据更准确

## ⚠️ 风险提示

### 方案A（稳健型）

- ✅ **优点**：数据准确，无预测误差
- ⚠️ **缺点**：数据延迟1-4天（周一到周四）
- 📊 **适用**：中长期投资

### 方案B（激进型）

- ✅ **优点**：数据更及时，能捕捉本周趋势
- ⚠️ **缺点**：引入预测误差，风险较高
- 📊 **适用**：短线交易

**重要提示**：
- 所有分析和建议仅供学习参考，不构成投资建议
- 方案B的拟合数据存在预测误差，请谨慎使用
- 建议结合其他分析工具和市场信息综合判断
